# 第6章 全场景工具链——从创意到交付的AI工作流

单个工具再好，不如串成链路。这章把前5章的工具打通，构建从创意到交付的完整AI工作流。

我是怕浪猫，AI提效工具系列第6章。前面5章逐个介绍了工具，这章做整合。

## 6.1 需求分析：不同场景的工具选型决策树

**选型决策树**

```
你的需求是什么？
├── 内容创作
│   ├── 写文章 → Ollama+OpenWebUI + AnythingLLM
│   ├── 做图片 → ComfyUI/WebUI + SDXL/FLUX
│   ├── 做视频 → ComfyUI + SVD/CogVideoX + Whisper-WebUI
│   └── 多语言 → 沉浸式翻译
├── 办公效率
│   ├── 知识库 → FastGPT（通用）/ RAGFlow（复杂文档）
│   ├── AI应用 → Dify（工作流编排）
│   ├── 自动化 → n8n（系统集成）
│   └── 文档对话 → AnythingLLM
├── 开发提效
│   ├── 代码补全 → Continue + 本地模型
│   ├── 代理编程 → Cline + DeepSeek
│   ├── 本地推理 → LocalAI / Ollama
│   └── 代码模型 → DeepSeek-Coder / Qwen2.5-Coder
└── 全栈AI
    ├── 对话平台 → OpenWebUI / LibreChat
    ├── 模型运行 → Ollama / LocalAI
    └── 工作流 → Dify + n8n
```

**选型核心原则**

| 原则 | 说明 |
|------|------|
| 先用再买 | 开源工具先试，不够再上商业方案 |
| 少即是多 | 3个精通的工具 > 10个浅尝的工具 |
| 数据为王 | 模型可以换，数据积累不能丢 |
| 链路思维 | 单点优化有限，链路优化才是质变 |
| 隐私分层 | 敏感数据本地，非敏感用API |

## 6.2 内容创作链路：写作+配图+视频一条龙

**完整链路设计**

```
选题策划 → 资料调研 → 初稿撰写 → 配图制作 → 视频化 → 多语言发布
```

**各环节工具与职责**

| 环节 | 工具 | 输入 | 输出 | 耗时 |
|------|------|------|------|------|
| 选题策划 | OpenWebUI + Qwen | 关键词/热点 | 选题清单 | 10分钟 |
| 资料调研 | AnythingLLM | 参考文档 | 摘要+要点 | 15分钟 |
| 初稿撰写 | OpenWebUI + DeepSeek | 大纲+要点 | 初稿 | 30分钟 |
| 配图制作 | ComfyUI + FLUX | 文章主题 | 3-5张配图 | 20分钟 |
| 视频化 | ComfyUI + SVD | 配图→动图 | 短视频素材 | 30分钟 |
| 字幕 | Whisper-WebUI | 音频/文本 | SRT字幕 | 10分钟 |
| 多语言 | 沉浸式翻译 | 中文成品 | 英文/日文版 | 15分钟 |
| 合计 | - | - | - | 约2小时 |

**传统方式对比**

| 环节 | 传统方式 | AI链路 | 提升幅度 |
|------|---------|--------|---------|
| 选题策划 | 1小时 | 10分钟 | 83% |
| 资料调研 | 2小时 | 15分钟 | 87% |
| 初稿撰写 | 4小时 | 30分钟 | 87% |
| 配图制作 | 1小时 | 20分钟 | 67% |
| 视频化 | 3小时 | 30分钟 | 83% |
| 多语言 | 3小时 | 15分钟 | 92% |
| 合计 | 14小时 | 2小时 | 86% |

**链路优化要点**

| 要点 | 说明 |
|------|------|
| 模板化 | 把每个环节的提示词保存为模板 |
| 批量化 | 配图用ComfyUI API批量生成 |
| 中间件 | 用n8n串联各环节，减少手动切换 |
| 版本管理 | 用Git管理所有中间产物 |
| 反馈循环 | 发布后用AnythingLLM分析评论反馈 |

## 6.3 办公自动化链路：文档处理+知识库+流程自动化

**链路设计**

```
文档输入 → 自动分类 → 知识库入库 → 智能检索 → 自动执行 → 结果交付
```

**各环节工具**

| 环节 | 工具 | 职责 |
|------|------|------|
| 文档输入 | n8n Webhook | 接收邮件/上传/API |
| 自动分类 | n8n + OpenAI节点 | 按类型/紧急度分类 |
| 知识库入库 | FastGPT API | 自动分段+向量化 |
| 深度解析 | RAGFlow | 处理表格/公式等复杂内容 |
| 智能检索 | Dify Chatflow | 对话式检索+推理 |
| 自动执行 | n8n | 发邮件/建工单/更新数据库 |
| 结果交付 | n8n → 飞书/邮件/Slack | 多渠道通知 |

**实战场景：合同审核自动化**

| 步骤 | 工具 | 操作 | 耗时 |
|------|------|------|------|
| 1 | n8n | 收到合同邮件→自动提取附件 | 1分钟 |
| 2 | RAGFlow | 解析合同→提取关键条款 | 2分钟 |
| 3 | Dify | AI审核→比对标准模板→标记风险 | 3分钟 |
| 4 | n8n | 生成审核报告→发送给法务 | 1分钟 |
| 5 | FastGPT | 合同存入知识库→可后续查询 | 1分钟 |
| 合计 | - | - | 8分钟 |

> 传统合同审核：人工阅读30-60分钟。AI链路：8分钟完成初筛，法务只看AI标记的风险点。

## 6.4 开发提效链路：代码生成+审查+部署

**链路设计**

```
需求分析 → 代码生成 → 代码审查 → 测试 → 部署 → 监控
```

**各环节工具**

| 环节 | 工具 | 职责 |
|------|------|------|
| 需求分析 | OpenWebUI + DeepSeek-R1 | 需求拆解、技术方案 |
| 代码生成 | Cline + DeepSeek-Coder | 自主编写代码 |
| 代码补全 | Continue + Qwen2.5-Coder | 实时补全 |
| 代码审查 | Continue Chat | 选中代码→AI审查 |
| 测试生成 | Continue Actions | 自动生成单元测试 |
| 文档生成 | Continue Actions | 自动生成注释和文档 |
| CI/CD | n8n + Git | 自动化部署流程 |

## 6.5 成本与隐私：开源vs闭源的长期TCO对比

**三年总持有成本（TCO）对比**

| 方案 | 初始成本 | 年运营成本 | 三年TCO | 隐私 |
|------|---------|-----------|---------|------|
| 全商业方案 | $0 | $1200-3600 | $3600-10800 | 云端 |
| 全开源本地 | $2000（硬件） | $500（电费） | $3500 | 完全本地 |
| 混合方案 | $1000 | $300-800 | $1900-3400 | 分层 |
| 开源+API | $0 | $200-600 | $600-1800 | 部分云端 |

**混合方案推荐**

| 场景 | 方案 | 理由 |
|------|------|------|
| 敏感代码 | 本地模型 | 数据不外泄 |
| 日常补全 | 本地Qwen2.5-Coder | 免费且够用 |
| 复杂重构 | DeepSeek API | 便宜且强大 |
| 图片生成 | 本地FLUX.1 schnell | 一次部署长期用 |
| 文档翻译 | 本地Ollama | 隐私保护 |
| 知识库 | 本地FastGPT | 数据自主 |
| 自动化 | 本地n8n | 系统集成 |

> 最优策略：80%用本地开源工具，20%按需调API。三年省下至少$5000，还拿到了数据主权。
