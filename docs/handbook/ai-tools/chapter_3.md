# 第3章 AI写作与内容生成——开源写作助手盘点

不用翻墙，不用付费，5个开源工具搭建你的私有AI写作助手。

我是怕浪猫，AI提效工具系列第3章。上两章聊了图片和视频，这章讲文字。

## 3.1 Ollama+OpenWebUI：本地大模型对话平台

**Ollama：本地运行大模型的极简方案**

Ollama把复杂的大模型部署简化到一行命令。

| 特性 | 说明 |
|------|------|
| 安装方式 | 一键安装（Mac/Windows/Linux） |
| 模型管理 | ollama pull/ollama run |
| 模型格式 | GGUF量化格式 |
| API兼容 | OpenAI兼容API |
| 模型库 | Llama、Qwen、DeepSeek、Mistral等 |

**常用模型推荐**

| 模型 | 参数量 | 显存需求 | 写作能力 | 中文能力 |
|------|--------|---------|---------|---------|
| Qwen2.5-7B | 7B | 6GB | 好 | 优秀 |
| DeepSeek-R1-7B | 7B | 6GB | 好 | 优秀 |
| Llama3.1-8B | 8B | 7GB | 优秀 | 中 |
| Qwen2.5-14B | 14B | 10GB | 很好 | 优秀 |
| DeepSeek-R1-14B | 14B | 10GB | 很好 | 优秀 |
| Mistral-7B | 7B | 6GB | 好 | 弱 |

> 中文写作首选Qwen和DeepSeek。7B模型在4-bit量化下只需6GB显存，集显笔记本也能跑。

**OpenWebUI：给Ollama套上ChatGPT界面**

| 功能 | 说明 |
|------|------|
| 多模型切换 | 下拉选择不同模型 |
| 对话管理 | 多会话、历史记录 |
| 文档对话 | 上传PDF/Word直接提问 |
| 联网搜索 | 集成搜索增强回答 |
| 提示词库 | 保存常用提示词模板 |
| 多用户 | 支持团队使用 |
| RAG | 内置检索增强生成 |

**3步部署**

| 步骤 | 命令/操作 | 说明 |
|------|-----------|------|
| 1. 装Ollama | ollama pull qwen2.5:7b | 下载模型 |
| 2. 装OpenWebUI | docker run -d -p 3000:8080 ... | Docker一行启动 |
| 3. 访问 | http://localhost:3000 | 浏览器使用 |

## 3.2 AnythingLLM：文档对话与知识管理一体化

AnythingLLM把"上传文档→AI问答"这个过程做到了极致。

**核心能力**

| 功能 | 说明 |
|------|------|
| 文档对话 | PDF/Word/Excel/PPT直接提问 |
| 多模型支持 | OpenAI/Claude/Ollama/LocalAI |
| 工作空间 | 按项目分类管理文档 |
| Agent能力 | 可执行网页搜索、文档操作 |
| 向量数据库 | 内置LanceDB，无需额外配置 |
| 桌面版 | Mac/Windows/Linux客户端 |

**AnythingLLM vs OpenWebUI文档对话**

| 维度 | AnythingLLM | OpenWebUI |
|------|-------------|-----------|
| 定位 | 文档知识管理 | 通用对话平台 |
| 文档解析 | 强（复杂表格/图表） | 中 |
| 多文档 | 工作空间分类管理 | 简单列表 |
| Agent | 支持 | 不支持 |
| 部署 | 桌面版/Docker | Docker |
| 适合 | 企业知识库 | 个人对话 |

> 如果你主要需求是"跟文档对话"，AnythingLLM体验更好。如果还需要通用聊天、提示词管理等，OpenWebUI更全面。

## 3.3 LibreChat：多模型对话与多端同步

LibreChat是另一个开源的ChatGPT替代品，最大亮点是多端同步。

**核心特性**

| 功能 | 说明 |
|------|------|
| 多模型 | OpenAI/Claude/Gemini/Ollama/本地模型 |
| 多端同步 | 同一账号跨设备同步对话 |
| Artifacts | 类似Claude的Artifacts功能 |
| 插件系统 | 搜索、代码执行、图片生成 |
| 对话分支 | 从任意消息分叉新对话 |
| 书签 | 保存重要对话片段 |
| 导入导出 | 对话数据可迁移 |

**3款对话工具对比**

| 维度 | OpenWebUI | AnythingLLM | LibreChat |
|------|-----------|-------------|-----------|
| 定位 | 本地模型对话 | 文档知识管理 | 多模型对话 |
| 多端同步 | 否 | 桌面版 | 是 |
| 文档对话 | 基础 | 强 | 基础 |
| 插件 | 少 | Agent | 丰富 |
| 部署难度 | 低 | 低 | 中 |
| 适合 | 个人本地 | 企业知识库 | 团队协作 |

> 三个工具不是互斥的。怕浪猫自己同时用OpenWebUI做日常对话，AnythingLLM管文档，LibreChat做多模型对比。

## 3.4 沉浸式翻译：开源AI翻译工具

沉浸式翻译（Immersive Translate）是2025年最火的开源翻译工具之一。

**核心功能**

| 功能 | 说明 |
|------|------|
| 网页翻译 | 浏览器实时双语翻译 |
| PDF翻译 | 保持排版的PDF翻译 |
| 视频字幕 | YouTube等平台实时字幕翻译 |
| EPUB翻译 | 电子书双语阅读 |
| 多引擎 | OpenAI/Claude/DeepL/Google/本地模型 |
| 自定义 | 可配置提示词和翻译风格 |

**翻译引擎对比**

| 引擎 | 质量 | 速度 | 成本 | 隐私 |
|------|------|------|------|------|
| OpenAI GPT-4o | 优秀 | 中 | 按Token | 云端 |
| Claude | 优秀 | 中 | 按Token | 云端 |
| DeepL | 很好 | 快 | 按字符 | 云端 |
| Google | 好 | 快 | 免费 | 云端 |
| 本地Ollama | 好 | 中 | 免费 | 本地 |

> 隐私敏感的文档用本地Ollama翻译，日常网页用Google或DeepL，重要文档用GPT-4o。沉浸式翻译支持按场景自动切换引擎。

**安装方式**

| 平台 | 安装方式 |
|------|---------|
| Chrome | Chrome Web Store搜索安装 |
| Firefox | Add-ons搜索安装 |
| 油猴脚本 | Tampermonkey脚本安装 |
| 桌面版 | 下载客户端（Mac/Windows） |

## 3.5 实战：搭建私有AI写作助手

把前面4个工具串起来，搭建一个完整的AI写作工作流。

**架构设计**

```
需求输入 → Ollama+OpenWebUI（创意生成）
    → AnythingLLM（素材检索+事实核查）
    → 沉浸式翻译（多语言版本）
    → 输出成品
```

**各环节工具与职责**

| 环节 | 工具 | 职责 |
|------|------|------|
| 创意生成 | OpenWebUI + Qwen2.5 | 头脑风暴、大纲生成 |
| 初稿撰写 | OpenWebUI + DeepSeek-R1 | 正文写作 |
| 素材检索 | AnythingLLM | 从知识库找支撑材料 |
| 事实核查 | AnythingLLM + 文档 | 验证关键数据 |
| 多语言 | 沉浸式翻译 | 英文/日文版本 |
| 润色 | OpenWebUI + 自定义提示词 | 风格优化 |

**提示词模板示例**

| 场景 | 提示词模板 |
|------|-----------|
| 大纲生成 | "请为以下主题生成一个{N}部分的系列文章大纲，每部分包含3个要点：{主题}" |
| 初稿撰写 | "请根据以下大纲撰写正文，风格{风格}，字数约{字数}，目标读者是{读者}：{大纲}" |
| 风格润色 | "请将以下文字改写为{风格}风格，保持信息不变，增强可读性：{原文}" |
| 事实核查 | "请检查以下陈述是否准确，标注不确定的部分：{陈述}" |

**效率提升数据**

| 环节 | 传统方式 | AI辅助 | 提升幅度 |
|------|---------|--------|---------|
| 大纲构思 | 30分钟 | 5分钟 | 83% |
| 初稿撰写 | 3小时 | 40分钟 | 78% |
| 素材查找 | 1小时 | 10分钟 | 83% |
| 多语言翻译 | 2小时 | 15分钟 | 87% |
| 润色修改 | 1小时 | 20分钟 | 67% |
| 合计 | 7.5小时 | 1.5小时 | 80% |

> AI写作不是让AI替你写，而是让AI做80%的基础工作，你专注20%的创意和判断。
