## 2.1 AI视频生成模型：SVD、CogVideoX、Mochi 1、LTX-Video

2025年开源AI视频生成领域，4个模型值得关注：

**Stable Video Diffusion (SVD)**

Stability AI出品，基于Stable Diffusion架构扩展到时序维度。

| 参数 | 说明 |
|------|------|
| 输入 | 单张图片（图生视频） |
| 输出 | 14-25帧，576x1024 |
| 显存需求 | 12GB起 |
| 帧率 | 约6fps（可插帧到24fps） |
| 许可证 | Community License |
| 优势 | 与SD生态兼容，图生视频效果好 |

**CogVideoX**

智谱AI出品，支持文生视频和图生视频，中文理解能力强。

| 参数 | 说明 |
|------|------|
| 输入 | 文本/图片 |
| 输出 | 6秒，720x480 |
| 显存需求 | 8GB（2B模型）/ 16GB（5B模型） |
| 许可证 | Apache 2.0（2B）/ 每日200次（5B） |
| 优势 | 中文提示词理解好，开源协议友好 |

**Mochi 1**

Genmo出品，2025年开源视频模型中运动质量最好之一。

| 参数 | 说明 |
|------|------|
| 输入 | 文本 |
| 输出 | 5.4秒，848x480 |
| 显存需求 | 4 H100 GPU（完整版）/ 1 GPU（量化版） |
| 许可证 | Apache 2.0 |
| 优势 | 运动流畅度高，物理逼真度好 |

**LTX-Video**

Lightricks出品，主打快速生成。

| 参数 | 说明 |
|------|------|
| 输入 | 文本/图片 |
| 输出 | 5秒，768x512 |
| 显存需求 | 12GB |
| 生成速度 | 30秒生成5秒视频（A100） |
| 许可证 | 开源 |
| 优势 | 速度快，可在单卡上运行 |

**4模型横评**

| 维度 | SVD | CogVideoX | Mochi 1 | LTX-Video |
|------|-----|-----------|---------|-----------|
| 生成方式 | 图生视频 | 文/图生视频 | 文生视频 | 文/图生视频 |
| 视频长度 | 2-4秒 | 6秒 | 5.4秒 | 5秒 |
| 运动质量 | 中 | 中 | 高 | 中高 |
| 画面质量 | 高 | 中高 | 中高 | 中 |
| 上手难度 | 低 | 低 | 中 | 低 |
| 硬件门槛 | 中 | 低 | 高 | 中 |
| 中文支持 | 否 | 是 | 否 | 否 |

> 选视频模型像选相机：SVD适合从静态图"激活"视频，CogVideoX适合中文场景，Mochi 1追求运动质量，LTX-Video追求速度。

## 2.2 ComfyUI视频工作流：AnimateDiff与SVD实战

ComfyUI不仅能做图片，视频工作流同样强大。

**AnimateDiff**

AnimateDiff是一个让SD模型生成动画的运动模块，原理是在现有SD模型上插入时序层。

| 特性 | 说明 |
|------|------|
| 基础模型 | SD1.5/SDXL |
| 输出 | 16帧动画，可循环 |
| 显存 | 8GB起 |
| 风格 | 继承底模风格 |
| 适用 | 动画风格短视频、循环GIF |

**SVD工作流**

SVD在ComfyUI中的典型工作流：

| 步骤 | 节点 | 说明 |
|------|------|------|
| 1 | Load Image | 加载起始图片 |
| 2 | SVD Image Encoder | 编码图片特征 |
| 3 | SVD Sampler | 采样生成视频帧 |
| 4 | VAE Decode | 解码为像素帧 |
| 5 | Video Combine | 合并为MP4/GIF |

**ComfyUI视频工作流效率技巧**

| 技巧 | 效果 |
|------|------|
| 用LTX-Video替代SVD | 生成速度提升3倍 |
| 先低分辨率生成再超分 | 显存节省40% |
| 用ControlNet控制运动 | 运动可预测性大幅提升 |
| 批量生成用API调度 | 无人值守批量产出 |
| AnimateDiff用循环帧 | 生成无缝循环动画 |

## 2.3 AutoCut：用AI自动剪辑视频的开源神器

视频剪辑最耗时的是什么？找素材、剪片段、加字幕。AutoCut用AI自动化这个过程。

**AutoCut工作原理**

| 输入 | AI处理 | 输出 |
|------|--------|------|
| 视频文件 | Whisper转录 | 带时间码的文字稿 |
| 文字稿 | LLM分析关键段落 | 自动剪辑决策 |
| 剪辑决策 | FFmpeg执行切割 | 输出剪辑后视频 |

**AutoCut vs 传统剪辑**

| 维度 | AutoCut | 手动剪辑（Premiere/达芬奇） |
|------|---------|---------------------------|
| 转录 | 自动（Whisper） | 手动或第三方工具 |
| 找片段 | AI标记关键内容 | 手动浏览全部素材 |
| 剪辑 | 选文字段即剪 | 手动切割拼接 |
| 字幕 | 自动生成 | 手动输入 |
| 效率 | 10分钟素材→5分钟出片 | 10分钟素材→1小时出片 |
| 精度 | 中 | 高 |
| 适合 | 口播/访谈/会议 | 精品内容创作 |

**安装与使用**

```
# 安装
pip install autodoc-mp4

# 基本用法
autocut video.mp4  # 自动转录+剪辑
autocut video.mp4 --lang zh  # 指定中文
```

> AutoCut不是要替代专业剪辑师，而是把"从素材到粗剪"这个最无聊的环节自动化。省下的时间用来做创意。

## 2.4 Whisper-WebUI：开源语音转文字与自动字幕

Whisper是OpenAI开源的语音识别模型，Whisper-WebUI给它套了一个网页界面。

**三种转录引擎对比**

| 引擎 | 速度 | 显存 | 精度 | 推荐场景 |
|------|------|------|------|---------|
| 标准Whisper | 标准 | 中 | 最高 | 专业场景 |
| faster-whisper | 快5倍 | 低 | 高 | 日常使用 |
| insanely-fast-whisper | 极速 | 最低 | 中高 | 批量处理 |

**Whisper-WebUI核心功能**

| 功能 | 说明 |
|------|------|
| 文件转录 | 上传音频/视频文件转文字 |
| YouTube转录 | 输入链接直接转录 |
| 麦克风实时转录 | 实时语音转文字 |
| 说话人识别 | 区分不同说话人 |
| 背景音乐分离 | 分离人声和BGM |
| 多语言支持 | 99种语言自动检测 |
| 输出格式 | TXT/SRT/VTT/JSON |

**部署方式**

| 方式 | 命令 | 适合 |
|------|------|------|
| Docker | docker compose up | 推荐方式 |
| Windows | Install.bat | 非技术用户 |
| Linux/Mac | ./Install.sh | 命令行用户 |
| 访问地址 | http://localhost:7860 | 浏览器使用 |

**Whisper模型选择指南**

| 模型 | 磁盘 | 内存 | 中文精度 | 推荐场景 |
|------|------|------|---------|---------|
| tiny | 75MB | 125MB | 低 | 快速预览 |
| base | 142MB | 210MB | 中低 | 测试 |
| small | 466MB | 600MB | 中 | 日常使用 |
| medium | 1.5GB | 2.1GB | 高 | 专业字幕 |
| large | 2.9GB | 3.3GB | 最高 | 精确转录 |

> 中文转录建议至少用small模型，medium是性价比最高的选择。large在长音频上精度优势明显。

## 2.5 视频后处理工具链：插帧、超分与编码

AI生成视频通常帧率低、分辨率低，需要后处理提升质量。

**插帧工具**

| 工具 | 原理 | 效果 | 速度 |
|------|------|------|------|
| RIFE | 光流+插值 | 好 | 快 |
| FILM | 大运动插值 | 很好 | 中 |
| AMT | 运动补偿 | 好 | 快 |
| ComfyUI-Framer | 节点化集成 | 好 | 中 |

**超分辨率工具**

| 工具 | 输入 | 输出 | 特点 |
|------|------|------|------|
| Real-ESRGAN | 480p视频 | 4K视频 | 通用性强 |
| GFPGAN | 人脸视频 | 高清人脸 | 人脸增强 |
| CodeFormer | 低质量视频 | 修复+增强 | 极端质量恢复 |
| Topaz Video AI | 任意视频 | 8K | 商业工具（对比） |

**编码与输出**

| 编码器 | 压缩率 | 速度 | 兼容性 |
|--------|--------|------|--------|
| H.264 (x264) | 标准 | 快 | 所有设备 |
| H.265 (x265) | 高50% | 中 | 现代设备 |
| AV1 | 最高 | 慢 | 新设备 |
| ProRes | 无损 | 快 | 后期制作 |

**完整后处理工作流**

```
AI生成视频(576p/6fps)
    → RIFE插帧(6fps→24fps)
    → Real-ESRGAN超分(576p→4K)
    → x265编码输出
    → 成品(4K/24fps)
```

| 步骤 | 工具 | 耗时(10秒视频) |
|------|------|---------------|
| 插帧 | RIFE | 约30秒 |
| 超分 | Real-ESRGAN | 约2分钟 |
| 编码 | x265 | 约1分钟 |
| 合计 | - | 约4分钟 |

> 10秒低分辨率视频，4分钟后变成4K高清。这就是开源工具链的威力。
